# All Endings

This file defines ending philosophy, not final ending scripts.

## Ending Philosophy

Endings should reflect identity, memory, emotional connection, and what the player understood. They should not be simple good, bad, and secret labels.

## Required Connections

Endings may depend on:

- Identity percentage.
- Memory fragments recovered.
- Journal state.
- Family connections preserved or lost.
- Puzzle truths discovered.
- Replay-only understanding.
- Future Director decisions.

## Replay Meaning

The first playthrough should feel complete enough to matter. The second playthrough should make the player reinterpret the first ending.

## Boundaries

Do not finalize ending names or exact outcomes without explicit story-phase approval. Do not add endings that contradict the Building's identity-consuming nature.

## Open Design Questions

- How many endings should exist?
- Can the player truly escape?
- Is preserving memory always the same as winning?
