# Memory System

Memories are collectible fragments of identity, not coins.

## Purpose

Memories should restore, reveal, corrupt, or reframe parts of the player's self. They may affect Journal state, Identity percentage, dialogue, endings, and future Director decisions.

## Design Rules

- A memory fragment must have emotional or narrative value.
- Collection should feel intimate, not like scavenging currency.
- Missing memories may appear as `???`.
- Replay may change what a memory means.

## Future Integration

Observation Engine can record memory interactions. Directors may react to memory recovery, avoidance, or loss. Save systems must preserve meaningful memory state.

## Open Design Questions

- Can the player reject a memory?
- Are some memories false but still emotionally true?
