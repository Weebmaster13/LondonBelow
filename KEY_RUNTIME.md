# Key Runtime

Key Runtime treats keys as data.

Definitions support single-use keys, reusable keys, master keys, party-shared keys, objective reward keys, puzzle reward keys, and future generated keys.

Key use validates that the player has the key and that the key can unlock the target. Single-use keys are consumed through Inventory Runtime.

The runtime does not create key models, final pickup UI, Chapter 1 placement, or physical door movement.
