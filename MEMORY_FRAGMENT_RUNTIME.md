# Memory Fragment Runtime

`MemoryFragmentRuntime` owns server-authoritative memory fragment unlock truth.

Memory Fragments are data schemas, not final memories. They support future replay meaning, identity interpretation, and narrative systems without writing canon in this foundation phase.

## Rules

- Server owns memory fragment truth.
- Duplicate fragment IDs reject per profile.
- Fragment metadata must be serializable and safe.
- Fragments must not contain final story dialogue, cutscene commands, UI commands, Workspace references, or Monster AI execution.

## Future Work

Future content phases may attach authored memories to approved schemas. Future UI may present unlocked fragments only after server approval.
