# Object Runtime

Object Runtime is the base data model for future interactive objects.

Every object definition includes a stable id, object kind, owner system, allowed states, initial state, interaction permissions, dependencies, observations, Director hooks, diagnostics, snapshot coverage, and future save metadata.

Supported future object kinds include doors, drawers, cabinets, levers, valves, switches, books, notes, paintings, statues, mirrors, clocks, bells, windows, ropes, chains, generators, fuse boxes, elevators, cranks, pressure plates, pedestals, artifacts, safes, chests, gates, secret passages, hidden walls, and custom objects.

The runtime does not create physical models, animations, final UI, Chapter 1 scripts, copied puzzles, or Workspace mutation.
